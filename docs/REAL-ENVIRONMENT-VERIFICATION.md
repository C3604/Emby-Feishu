# 真实环境验证清单

本清单面向**完全不懂开发**的用户。请在一台真实的 Emby Server 4.9.5.0 上按顺序操作。
每一步都写明了「操作」「预期结果」和「失败时查看什么」。

> 准备工作：一个能收消息的飞书群 + 该群自定义机器人的 Webhook 地址；一个可播放的视频；最好有两台能登录 Emby 的设备（手机 + 电脑）。

---

## A. 安装与加载

### 1. 备份旧插件
- 操作：如果 Emby 插件目录里已有旧的 `EmbyFeishu.dll`，先复制一份到别处备份。
- 预期：有一个旧文件备份。
- 失败时：若从未安装过，跳过本步。

### 2. 停止 Emby Server
- 操作：停止 Emby 服务（Windows 服务管理器 / `systemctl stop emby-server` / `docker stop emby`）。
- 预期：Emby 后台无法访问。

### 3. 替换插件 DLL
- 操作：把新的 `src/EmbyFeishu/bin/Release/EmbyFeishu.dll` 复制到 Emby 插件目录，覆盖旧文件。**只复制这一个文件。**
- 预期：插件目录中 `EmbyFeishu.dll` 是新版本（1.4.0.0）。
- 失败时：确认没有误传 `deps.json`、`pdb` 或任何 `MediaBrowser.*.dll`。

### 4. 启动 Emby Server
- 操作：启动 Emby 服务。
- 预期：Emby 后台可正常访问。

### 5. 确认插件显示
- 操作：Emby 后台 → 插件。
- 预期：看到「Emby 飞书通知」，版本 1.4.0.0。
- 失败时：查看 Emby 日志（路径见 DEPLOYMENT.md 第八节），搜索 `EmbyFeishu` 与 `error`。

---

## B. 配置与即时生效

### 6. 打开配置页
- 操作：点击「Emby 飞书通知」。
- 预期：配置页正常显示，能看到「启用插件」「飞书 Webhook 地址」等选项。
- 失败时：强制刷新浏览器（Ctrl+Shift+R）；查看日志。

### 7. 填写 Webhook 并先只开「开始播放」
- 操作：粘贴 Webhook 地址；勾选「启用插件」；打开「通知播放开始」；**关闭**「通知播放停止/暂停/恢复」；保存。
- 预期：保存成功，无报错。
- 失败时：若提示地址错误，检查是否以 `https://` 开头。

### 8. 用测试推送验证配置
- 操作：勾选「发送测试通知」→ 保存。
- 预期：飞书群收到一条「测试」消息；配置页「上次测试结果」显示 ✅。
- 失败时：看「上次测试结果」的错误文字；查看日志中 `[EmbyFeishu] 测试推送`。

---

## C. 播放事件

### 9. 播放视频（开始通知）
- 操作：用测试账号播放一个视频。
- 预期：飞书群收到**一条**「开始播放」通知（约数秒内）。
- 失败时：确认「仅通知视频播放」与视频类型匹配；查看日志 `通知已入队` / `通知发送成功`。

### 10. 检查开始通知只有一条
- 操作：观察飞书群。
- 预期：同一次播放只有一条开始通知，不重复。
- 失败时：记录重复条数与时间，查看日志。

### 11. 开启暂停/恢复/停止通知
- 操作：回到配置页，打开「通知播放停止/暂停/恢复」，保存（无需重启）。
- 预期：保存成功。

### 12. 暂停并保持约 10 秒（暂停去重）
- 操作：暂停播放，保持 10 秒。
- 预期：飞书群只收到**一条**「暂停播放」通知（不因进度刷新重复）。
- 失败时：若出现多条，记录条数，查看日志。

### 13. 恢复播放（恢复去重）
- 操作：继续播放。
- 预期：只收到**一条**「恢复播放」通知。

### 14. 停止播放（停止通知）
- 操作：停止播放。
- 预期：只收到**一条**「停止播放」通知。

---

## D. 多会话与容错

### 15. 两台设备同时播放
- 操作：手机和电脑分别播放不同视频。
- 预期：两条独立的开始通知；分别暂停/恢复互不影响。
- 失败时：若一台的操作影响了另一台的通知，记录现象并查看日志。

### 16. 填写错误 Webhook 不影响播放
- 操作：把 Webhook 改成一个格式正确但无效的地址（如把末尾改几位），保存；再播放一个视频。
- 预期：视频**正常播放不卡顿**；飞书收不到消息；日志出现发送失败但无异常崩溃。
- 失败时：若播放受影响，立即记录并恢复正确 Webhook。

### 17. 【关键】日志中不含完整 Webhook
- 操作：打开 Emby 日志文件，搜索你的 Webhook 末尾 Token（`hook/` 后面那串）。
- 预期：**搜不到完整 Token**；只能看到类似 `open.feishu.cn/****xxxx` 的脱敏形式。
- 失败时：若日志出现完整地址，请记录该行并反馈（这是安全问题）。

---

## E. 持久化与产物

### 18. 修改配置不重启即时生效
- 操作：关闭「通知播放开始」，保存，再播放视频。
- 预期：不再收到开始通知（无需重启 Emby）。

### 19. 重启后配置保留
- 操作：重启 Emby Server，回到配置页。
- 预期：之前填写的 Webhook 和开关状态都还在。

### 20. 插件目录只有必要文件
- 操作：查看 Emby 插件目录。
- 预期：与 EmbyFeishu 相关的只有 `EmbyFeishu.dll`，没有其他 Emby DLL。

---

完成以上全部步骤且预期均满足，即可正式投入使用。任何一步出现异常，请保留 Emby 日志中 `[EmbyFeishu]` 相关行以便排查。
