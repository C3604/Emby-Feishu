# 测试文档

## 自测工具

项目包含一个独立的控制台自测工具，不依赖第三方测试框架。

### 运行自测

```bash
dotnet run --project tools/EmbyFeishu.SelfTest/EmbyFeishu.SelfTest.csproj
```

### 测试覆盖（共 150 项，全部通过）

**媒体标题格式化**：电影 / 剧集（含/无季集号）/ 空值降级。

**播放时间格式化**：HH:mm:ss / mm:ss / null / 0 / 负数。

**Webhook 脱敏**：移除完整 URL 与 Token、保留域名末四位、null/空安全。

**敏感信息脱敏器**：IP 隐藏/脱敏(IPv4 保前两段、IPv6 保前两组)/完整；设备 ID 隐藏/脱敏(末四位)/完整；异常消息移除 Windows/Unix 绝对路径且保留文件名；路径仅保留末段。

**用户过滤**：All / IncludeOnly / Exclude / 未知用户 / 忽略大小写。

**配置校验**：Webhook 空、非 HTTPS、合法飞书、非飞书域名不阻断、超时范围、用户名去重、里程碑解析（升序/去重/剔除越界与非法）。

**配置迁移**：默认 Custom/Text、旧播放开关默认保留、新字段安全默认、越界数值夹紧、架构版本提升。

**媒体类型分类**：Movie/Episode/Audio/Folder/CollectionFolder/Person/Genre 分类；文件夹与人物永不通知；预告片与音乐艺术家默认不通知；配置开关生效。

**文本格式化（默认 Custom 保持旧版外观）**：标题/用户/剧集/客户端/设备/时间；Custom 默认隐藏类型与播放方式；空字段省略。

**详细程度**：Simple/Standard/Detailed（技术字段受「显示敏感技术细节」开关控制）/Custom（尊重旧字段开关）。

**飞书卡片**：msg_type=interactive、header/elements 结构、严重程度配色映射、特殊字符不破坏结构。

**播放状态去重与多会话隔离**：Playing/Paused/Resumed 状态机、多会话互不干扰、会话键构建规则。

**进度里程碑与播放方式**：里程碑只触发一次、快进跨越只推最高、播放方式首次建基线不推仅真正改变才推、播放完成互斥标记。

**去重缓存**：窗口内重复抑制、不同键独立、窗口为 0 不去重。

**滑动窗口限流**：上限内放行、超限拒绝。

**通知策略**：普通限流、被抑制计数、安全事件豁免限流、登录失败短时去重防风暴。

**媒体库聚合**：少量逐条 + 同 ItemId 去重、超过上限转汇总、插件停止时安全刷新（Dispose）。

**测试推送配置**：标志与结果字段、Webhook URL 校验。

## 真实环境测试

自测无法模拟的场景需在真实 Emby 4.9.5.0 上验证，详见
[REAL-ENVIRONMENT-VERIFICATION.md](REAL-ENVIRONMENT-VERIFICATION.md)，要点：

1. 插件加载与配置页渲染/保存/即时生效。
2. 播放开始/停止/完成各只一次；暂停/恢复在高频进度下去重。
3. 里程碑、播放方式变化在真实播放中触发正确。
4. 登录失败推送不含密码；日志与消息不含完整 Webhook / Token / AccessToken。
5. 媒体库扫描时聚合生效、不产生消息风暴。
6. UserDataSaved 不因播放进度高频推送。
7. 任务失败摘要脱敏、无堆栈/绝对路径。
8. 无 Live TV 时插件仍正常加载。
9. 多设备并发互不干扰；飞书不可用不影响播放。
10. 卡片模式渲染正常；卡片失败回退文本仅一次。
