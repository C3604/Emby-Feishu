# 故障排查

## 插件未显示在 Emby 后台

1. **确认文件位置**：检查 `EmbyFeishu.dll` 是否在 Emby 的 `plugins` 目录中
2. **确认已重启**：上传插件后必须重启 Emby Server
3. **检查日志**：在 Emby 日志中搜索 `EmbyFeishu`，看是否有加载错误
4. **检查版本**：确保你的 Emby Server 版本是 4.9.5.0

## 配置页面打不开或报错

1. **检查 DLL 冲突**：确保 plugins 目录中没有多余的 MediaBrowser.*.dll 或 Emby.Web.GenericEdit.dll
2. **清除浏览器缓存**：Ctrl+Shift+R 强制刷新
3. **查看日志**：搜索 `[EmbyFeishu]` 查看错误信息

## 飞书群没有收到通知

### 检查清单

1. **插件是否启用**：在插件设置中确认"启用插件"已打开
2. **Webhook 地址是否正确**：
   - 必须以 `https://` 开头
   - 通常包含 `open.feishu.cn` 或 `larksuite.com`
   - 检查是否有多余的空格
3. **机器人是否仍在群中**：确认飞书群中的自定义机器人没有被移除
4. **事件开关**：确认"通知播放开始"等开关已打开
5. **仅视频模式**：如果你在播放音频，检查"仅通知视频播放"是否开启
6. **用户过滤**：检查"用户过滤模式"是否将当前用户排除了
7. **最短播放时长**：如果视频播放时间很短就停止了，可能被"最短播放秒数"过滤

### 查看日志

在 Emby 日志中搜索以下关键词：

- `[EmbyFeishu] 通知发送成功` — 正常发送
- `[EmbyFeishu] 通知发送失败` — 发送失败，查看后面的错误信息
- `[EmbyFeishu] 通知已入队` — 事件已识别并入队
- `[EmbyFeishu] 飞书业务错误` — 飞书返回了错误
- `[EmbyFeishu] Webhook 请求失败` — 网络请求失败

### 常见飞书错误

- **code=19021**：Webhook 地址无效或已过期，需要重新创建机器人
- **code=19024**：请求频率超限，等待几分钟后重试
- **code=9499**：机器人已被禁用

## Webhook 或配置保存后丢失 / 未生效

1. **确认点击了保存**：填写后必须点击页面底部的保存按钮，页面无报错才算成功。
2. **校验未通过会阻止保存**：Webhook 必须以 `https://` 开头；若启用了签名校验但未填密钥、或启用了关键词但关键词为空，会提示错误且不保存。
3. **签名密钥留空的含义**：签名密钥输入框留空表示“保持原有密钥不变”，不是清空。要更换密钥请输入新值再保存。
4. **保存即时生效**：本插件配置保存后立即生效，无需重启；若怀疑未生效，用「发送测试通知」验证。
5. **日志确认**：日志出现 `[EmbyFeishu] 配置已保存` 即表示已写入。

## 测试通知失败

1. 勾选「发送测试通知」并保存后，查看「上次测试结果」字段的文字：
   - `❌ 测试失败：Webhook 地址为空` — 先填写 Webhook。
   - `❌ 测试失败：飞书返回 code=…` — 按上文「常见飞书错误」处理。
   - `❌ 测试失败：<网络错误>` — 检查服务器到 `open.feishu.cn` 的网络与代理。
2. 日志搜索 `[EmbyFeishu] 测试推送` 查看详细过程。
3. 测试消息已应用与正常通知相同的关键词与签名装饰，可用于验证安全设置是否正确。

## 关键词相关错误（飞书拒收）

- 现象：飞书返回关键词不匹配类错误，或群里始终收不到消息，但日志显示已发送。
- 处理：
  1. 确认「启用自定义关键词」已开，且「自定义关键词」与飞书机器人「安全设置 → 自定义关键词」**完全一致**（区分大小写与空格）。
  2. 插件不会修改飞书端设置，两边必须手动保持一致。
  3. 关键词会自动追加到每条消息，无需在其他字段手动包含。

## 签名校验失败（飞书拒收）

- 现象：飞书返回签名不匹配（如 `sign match fail`）。
- 处理：
  1. 确认飞书机器人「安全设置」确实开启了「签名校验」，且插件也开启了「启用签名校验」。
  2. 确认填写的是**签名密钥**（签名校验页面提供），不是 Webhook 地址里的 Token。
  3. 若近期在飞书端重置过密钥，请在插件中重新输入新密钥并保存。
  4. 服务器时间需大致准确（签名含时间戳），偏差过大会导致校验失败。

## 收到重复消息

1. **暂停/恢复过多**：关闭「通知播放暂停」「通知播放恢复」，或增大「最短播放秒数」。
2. **确认只装了一份插件**：plugins 目录只应有一个 `EmbyFeishu.dll`，不要保留多版本副本。
3. **去重窗口**：插件对相同事件有短时去重；若仍偶发重复，记录时间与条数并查看日志 `通知已入队` 的频率。
4. **多设备并发**：不同设备的同类事件是彼此独立的，不属于重复。

## 通知太频繁

- 关闭"通知播放暂停"和"通知播放恢复"
- 增大"最短播放秒数"（默认 5 秒）
- 使用用户过滤，只通知特定用户

## 影响了 Emby 播放

正常情况下插件不应影响播放。如果出现异常：

1. 在插件设置中关闭"启用插件"并保存
2. 如果无法进入设置，直接删除 `EmbyFeishu.dll` 并重启 Emby
3. 将 Emby 日志中 `[EmbyFeishu]` 相关的内容提交给开发者排查
