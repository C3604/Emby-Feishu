# 飞书安全校验指南（FEISHU-SECURITY）

本文说明如何为 Emby 飞书通知插件配置飞书自定义机器人的安全校验功能。

## 一、添加飞书自定义机器人

1. 在飞书 PC 或移动端打开目标群聊。
2. 点击群组右上角的 **设置** → **群机器人**。
3. 点击 **添加机器人** → **自定义机器人**。
4. 设置机器人名称和头像后，点击 **添加**。
5. 复制生成的 **Webhook 地址**，粘贴到插件配置的「飞书连接」分组中。

官方参考文档：https://open.feishu.cn/document/client-docs/bot-v3/add-custom-bot

## 二、自定义关键词

### 2.1 在飞书中开启
1. 在「自定义机器人」设置页面找到 **安全设置**。
2. 选择 **自定义关键词**。
3. 输入一个关键词（例如 `Emby通知`），点击保存。
4. 飞书机器人将过滤所有群消息，只有包含该关键词的消息才会被推送。

### 2.2 在插件中配置
1. 打开 Emby 管理控制台 → **飞书通知**。
2. 进入「机器人安全校验」分组。
3. 打开 **启用自定义关键词** 开关。
4. 在 **自定义关键词** 输入框中填写与飞书设置中完全一致的关键词。
5. 点击保存。

### 2.3 注意事项
- 关键词必须与飞书群机器人安全设置中的自定义关键词**完全一致**，包括大小写。
- 插件不会自动修改飞书机器人配置，需要分别在飞书和插件中配置。
- 开启后，插件发送的每一条消息（文本/卡片/回退文本）都会在正文末尾携带关键词。
- 若消息正文已包含完全相同的关键词，不会重复追加。
- 关键词是飞书校验消息内容的一部分，必须属于用户可见的消息正文。

## 三、签名校验

### 3.1 在飞书中开启
1. 在「自定义机器人」设置页面找到 **安全设置**。
2. 选择 **签名校验**。
3. 飞书会生成一个 **签名密钥**（一串随机字符）。
4. 复制这个密钥。

**注意：签名密钥 ≠ Webhook 地址中的 Token。** Webhook URL 末尾的字符串是 Token，而签名密钥在安全设置中单独显示。

### 3.2 在插件中配置
1. 打开 Emby 管理控制台 → **飞书通知**。
2. 进入「机器人安全校验」分组。
3. 打开 **启用签名校验** 开关。
4. 在 **签名密钥** 输入框中粘贴飞书提供的密钥。
5. 点击保存。

### 3.3 密钥管理
- **留空保持原值**：如果已经配置过密钥，打开配置页面时密钥输入框为空（不暴露真实密钥）。不填新值直接保存会保持旧密钥不变。
- **输入新值替换**：在输入框中输入新密钥并保存，会用新值替换旧密钥。
- **手动清空并保存**：在输入框中手动清空内容并保存，将清除已存储的密钥。
- **启用签名但密钥为空**：保存会被阻止并提示"启用签名校验时，必须填写签名密钥"。
- **关闭签名校验时**：允许密钥为空。
- **密钥安全**：插件不会在配置页面中显示真实密钥，也不会在日志中记录完整密钥内容。

### 3.4 签名算法
插件按照飞书官方指南实现签名算法：

- 签名时间戳：当前 UTC Unix 秒
- 签名串：`timestamp + "\n" + secret`
- HMAC-SHA256：密钥为签名串的 UTF-8 字节，待签名内容为空字节数组
- 结果：Base64 编码（非 Base64Url）
- 每次发送（包括重试和卡片回退）都重新生成时间戳和签名

### 3.5 签名依赖
- **依赖服务器系统时间**：签名使用 UTC Unix 时间戳，请确保 Emby Server 所在机器的系统时间准确。时间偏差可能导致飞书拒绝请求。
- 如果发送失败并提示签名相关错误，请首先检查服务器系统时间是否与标准时间一致。

## 四、关键词与签名同时使用

自定义关键词和签名校验可以**同时启用**，两者不互斥：

- 同时启用时：消息正文包含关键词，请求顶层包含 timestamp 和 sign。
- 建议同时开启以获得最佳安全效果。

## 五、常见问题

### 消息发送失败
1. 检查 Webhook 地址是否正确。
2. 检查自定义关键词是否与飞书设置一致。
3. 检查签名密钥是否与飞书设置一致。
4. 检查服务器系统时间是否准确。
5. 尝试关闭所有安全设置测试基础连通性，再逐个开启安全功能。

### 签名验证失败
- 确认使用的是飞书签名密钥而非 Webhook Token。
- 确认密钥完全一致（无多余空格或换行）。
- 确认服务器系统时间与标准时间偏差在可接受范围内。
- 签名算法使用 HMAC-SHA256，密钥是 `timestamp + "\n" + secret`，不是直接使用 secret 作密钥。

### 日志信息安全
- 插件不会在 Emby 日志中输出完整的 Webhook URL（仅显示域名和末尾四位）。
- 插件不会输出完整的签名密钥。
- 插件不会输出完整的 sign 值。
- 插件不会输出包含安全信息的完整请求 JSON。
- 如怀疑日志泄露敏感信息，请检查 Emby 日志中的 `[EmbyFeishu]` 前缀条目。

## 六、配置文件安全

配置文件存储在 Emby 数据目录中。签名密钥以加密形式或原始形式存储（取决于 Emby 配置序列化机制）。建议：

- 限制 Emby 数据目录的文件系统访问权限。
- 不要在共享环境或版本控制中暴露配置文件。
- 使用 Emby 内置的用户访问控制限制管理控制台访问。
